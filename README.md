## ゲーム概要

灰色のもふもふ生き物「けだまる」を操作し、流れてくるアイテムを集めながら障害物を避けて 500m を目指すブラウザゲームです。

障害物に当たるたびに毛量が減り、0% になるとゲームオーバー。  
アイテムを取り続けてコンボを稼ぎ、ゴールまで駆け抜けよう！

## 主な機能

- **500m ゴール制** — 生き残って走りきるとゴール画面へ。道中は距離とともに難易度が上昇
- **3 段階レベルシステム** — 距離に応じて Lv.1→2→3 に突入、速度・障害物数・スポーン頻度が変化
- **背景のレベル連動** — Lv.1=昼の青空 / Lv.2=夕焼け / Lv.3=夜空（雲が星に変わる）
- **毛量システム** — 障害物に当たると毛が抜ける。毛量に応じてキャラの見た目が 5 段階で変化
- **コンボシステム** — 3 連続以上のアイテム取得でボーナス（3 連=+15, 4 連=+20…）
- **スクリーンシェイク** — 障害物ヒット時に画面が揺れる
- **スパークルエフェクト** — アイテム取得（黄）・ダメージ（赤）でパーティクルが飛び散る
- **ハイスコア保存** — localStorage でベストスコアを記録。タイトルと結果画面に表示

## けだまるの毛量ステージ

| 毛量 | 見た目 |
|------|--------|
| 80〜100% | もふもふ（90 本・隙間なし） |
| 50〜79%  | 普通（26 本） |
| 30〜49%  | 頭頂うすめ（上部 60° が薄い） |
| 10〜29%  | M 字（左右サイドのみ） |
| 0〜9%    | ほぼ丸（3 本だけ残る） |

## 学習ポイント

### JavaScript
- `requestAnimationFrame` によるゲームループと `dt`（デルタタイム）でフレームレートを吸収する仕組み
- Canvas API での図形・グラデーション・絵文字の描画
- 円と円の当たり判定（中心間距離 < 半径の和）
- `filter` で配列要素を管理（アイテム・障害物・パーティクル）
- `localStorage` によるスコア永続化
- `ctx.save() / ctx.restore()` で描画状態を分離したスクリーンシェイクの実装
- DPR（devicePixelRatio）対応で Retina ディスプレイでも高解像度に描画

### ゲーム設計
- `gamePhase`（title / playing / gameover / cleared）による状態管理
- データをテーブル（配列）で定義して拡張しやすくする設計（アイテム・障害物・レベル）
- ランダムタイマー方式のスポーンで毎回違うリズムを生む
- 難易度バランス調整（速度・スポーン頻度・障害物Y分布）

### Git / GitHub
- `git init` → `git add` → `git commit` → `git push` の基本フロー
- SSH キー設定（個人 GitHub アカウントの分離）
- README.md の整備
